Build an approval workflow in Power Automate — no code required

April 2026 · 8 min read

If your team still routes approval requests via email — forwarding chains, "did anyone see this?", and lost replies — you're not alone. It's one of the most common inefficiencies I see in small business M365 tenants. And it's also one of the easiest to fix.

Power Automate ships with a built-in approval connector. No code. No third-party tools. No additional cost if you're already on M365 Business Basic or above. Here's how to build a working approval workflow from scratch in about 30 minutes.

What we're building

A simple one-step approval flow triggered when someone submits a Microsoft Form. The approver gets an email with Approve/Reject buttons. The submitter gets notified of the outcome. Everything is logged automatically.

Prerequisites: Microsoft 365 Business Basic or above. You'll use Power Automate (cloud flows) and Microsoft Forms — both included in your subscription.

Step 1: Create your form

Go to forms.microsoft.com and create a new form. For this example, we'll build a purchase request form with three fields:

  • Requester name (text)
  • What are you requesting? (text, long answer)
  • Estimated cost (number)

Keep it simple. You can always add fields later. Save the form and note its name — you'll need it in the next step.

Step 2: Open Power Automate and create a new flow

Go to make.powerautomate.com. Click Create in the left sidebar, then choose Automated cloud flow. Give your flow a name like "Purchase Approval Workflow".

Search for "Microsoft Forms" in the trigger search box and select When a new response is submitted. Choose your form from the dropdown and click Create.

Step 3: Get the form response details

Click New step and add the Get response details action from Microsoft Forms. Set the Form ID to match your form, and set the Response ID to the dynamic value from your trigger.

This step pulls the actual answer data so you can use it in the approval email.

Step 4: Start an approval

Add another step: search for Approvals and select Start and wait for an approval. Configure it like this:

  • Approval type: Approve/Reject — First to respond
  • Title: "Purchase Request — [Requester Name]" (use the dynamic value from step 3)
  • Assigned to: your email address, or whoever should approve
  • Details: paste in the request details and cost from the form response

Step 5: Send outcome notifications

After the approval step, add a Condition to check whether the outcome equals "Approve". Then:

  • In the Yes branch: send an email to the requester saying their request was approved
  • In the No branch: send an email saying it was rejected, and include the approver's comments

Use the Send an email (V2) action from Office 365 Outlook for both branches.

Test it

Save your flow, then submit a test response through your form. You'll get an approval email within seconds. Approve or reject it, then check that the outcome email reaches the test requester address.

Tip: Check your flow's run history in Power Automate to troubleshoot any failed steps. Each action shows its inputs and outputs, which makes debugging straightforward.

What to build next

Once you've got this working, there are several natural extensions worth exploring:

  • Multi-stage approvals (manager first, then finance)
  • Log every request and its outcome to a SharePoint list
  • Escalation flows if the approver doesn't respond within 48 hours
  • A Power Apps front-end instead of a Form, for more control over the UI

None of these require code. They're all connectors available in Power Automate out of the box.
